# Rhyffle v2 — Discord & 팀 메시지 로그

> Discord 및 팀 메시지 중 spec에 영향을 주는 내용을 시간순(KST)으로 기록.
>
> ## 자동 기록 트리거
> 다음 중 하나에 해당하면 Claude는 이 파일에 자동 append 후 사용자에게 한 줄 보고:
> - 사용자 메시지에 "디코 메세지" / "디코" 가 명시된 경우
> - 메시지에 **김한울** 또는 **김은규** 의 발언이 인용된 경우
>
> ## 콘텐츠 필터
> **기획 또는 일정 관련 내용만 저장.** 잡담은 폐기.
> - ✅ 저장: spec 결정/제안/질문, 회의 일정, 마일스톤, 데드라인, 작업 분배, sprint 범위 변경, 작업에 영향 주는 외부 일정
> - ❌ 폐기: 잡담, 농담, 안부, 감정 표현, spec 무관 개인 의견
> - 애매하면 저장 쪽으로 기울이고 사용자에게 확인. 잡담 명백하면 "잡담이라 미기록" 한 줄만 보고.
>
> ## 기록 형식
> ```
> ## YYYY-MM-DD
>
> ### 화자 (HH:MM KST, 채널/스레드명)
> > [원문]
>
> **맥락**: ...
> **spec 영향**: ...
> **상태**: pending / Notion 반영 완료 / 폐기
> ```
>
> ## 위계
> - **Notion** = 정식 spec (source of truth)
> - **이 파일** = raw input — Notion 갱신 전 임시 spec 소스, 히스토리 추적용
> - 디코 내용이 spec에 영향 주면 → 가능한 빨리 사용자에게 "Notion에도 반영하시겠어요?" 확인

> @이재근 기존의 HoldNote가 json에서 이런 형식으로 나타났는데,

> {
>   "position": 614,     // 노트 등장 시점

>   "noteType": 2,       // HoldNote 종류 (0: 첫 처리, 1: 중간, 2: 마지막)
>   "count": 8,          // 연결된 HoldNote 번호 (count가 8인 노트들과 연결)

**맥락**: HoldNote JSON 스키마 변경 제안 — `noteType` 필드 제거, `count`만으로 묶음 처리 (게임 클라이언트에서 position 정렬 후 연결)
