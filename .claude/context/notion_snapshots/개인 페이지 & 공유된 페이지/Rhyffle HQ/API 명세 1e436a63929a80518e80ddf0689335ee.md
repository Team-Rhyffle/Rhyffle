## 클라이언트 수신

기본적으로 클라이언트가 수신하는 정보들은 json 형식으로 제공이 될 예정

각 부분별로 첨부된 json파일들은 수신받을 json의 서식에 대한 예시

### 계정 정보

1. 유저 정보
2. 유저 상태 정보
3. 유저 재화’
    
    [userCurrency.json](API%20%EB%AA%85%EC%84%B8/userCurrency.json)
    
    - user_id
        
        유저의 식별 번호 **int**
        
    - stardust, starpiece , antimatter
        
        카드 제작에 필요한 재료로 각각 **int**, UI 표시 필요
        
    - material
        
        그 외 재화 
        

### 카드 시스템

1. 덱 정보
    
    [deckInfo.json](API%20%EB%AA%85%EC%84%B8/deckInfo.json)
    
    - user_id
        
        유저의 식별 번호 **int**
        
    - decks
        
        유저의 덱 목록
        
        - deck_id
            
            덱의 식별 번호 **int**
            
        - deck_name
            
            덱 이름 **string**
            
        - deck_card_list
            
            덱에 들어갈 카드 식별 번호 60개의 **int 배열**
            
        - bookmared : 북마크한(즐겨찾기 같은 느낌) 덱으로 다른 덱보다 먼저 나열될 지 판단하는 **bool**
2. 유저 카드 정보
    
    [cardInfo2.json](API%20%EB%AA%85%EC%84%B8/cardInfo2.json)
    
    - user_id
        
        유저의 식별 번호 **int**
        
    - user_card_page
        
        유저의 카드가 많아질 경우를 대비하여 카드별로 페이지네이션을 함. 해당 변수는 현재 json 정보가 몇 번째 페이지인지 확인하는 용도의 **int**
        
    - cards_in_page
        
        현재 페이지에 있는 카드의 개수 **int**
        
    - cards
        
        카드 목록
        
        - card_id
            
            카드 식별 번호 **int**
            
        - card_baseid
            
            카드 기본 정보 식별 번호 **int**
            
        - pack_id
            
            팩 식별 번호 **int**
            
        - card_particle_id
            
            카드 파티클(카드 반짝임 같은 치장 관련) 식별 번호 **int**
            
        - ability_id
            
            일반 특수 능력 식별 번호 **int**
            
        - ability_level
            
            일반 특수 능력 레벨 **int**
            
        - unique_ability_level
            
            고유 특수 능력 레벨 **int**
            
        - bookmarked
            
            북마크한 카드인지 확인하는 **bool**
            
    1. 팩 정보
        
        [packInfo.json](API%20%EB%AA%85%EC%84%B8/packInfo.json)
        
        - packs
            
            팩 정보 목록
            
            - pack_id
                
                팩 식별 번호 **int**
                
            - pack_name
                
                팩 이름 **string**
                
    2. 일반 특수 능력 정보
        
        [abilityInfo.json](API%20%EB%AA%85%EC%84%B8/abilityInfo.json)
        
        - abilities
            
            일반 특수 능력 목록
            
            - ability_id
                
                일반 특수 능력 식별 번호 **int**
                
            - ability_name
                
                일반 특수 능력 이름 **string**
                
    3. 카드 파티클 정보
        
        [card_particle.json](API%20%EB%AA%85%EC%84%B8/card_particle.json)
        
        - card_particles
            
            카드 파티클 목록
            
            - card_particle_id
                
                카드 파티클 식별 번호 **int**
                
            - card_particle_name
                
                카드 파티클 이름 **string**
                
3. 카드 기본 정보
    
    [cardBaseInfo.json](API%20%EB%AA%85%EC%84%B8/cardBaseInfo.json)
    
    - card_base_page
        
        카드 기본 정보 페이지 번호 **int**
        
    - card_index
        
        카드 기본 정보 목록
        
        - card_baseid
            
            카드 기본 정보 식별 번호 **int**
            
        - card_suit
            
            카드 문양으로
            
            0 → 스페이드
            
            1 → 하트
            
            2 → 다이아몬드
            
            3 → 클로버
            
            위 값 중 하나를 가지는 **int**
            
        - card_rank
            
            카드의 숫자(1~13 ; 11, 12, 13은 각각 J, Q, K )를 가지는 **int**
            
        - card_rarity
            
            카드 희귀도 등급으로
            
            0 → Echo
            
            1 → Orbit
            
            2 → Ascendent
            
            3 → Arcana
            
            위 값 중 하나를 가지는 **int**
            
        - card_name
            
            카드 이름 **string**
            
        - unique_ability_id
            
            고유 특수 효과 식별 번호 **int**
            
        - collections
            
            컬랙션(카드군) 이름 **string**
            
    1. 고유 특수 능력 정보
        
        [uniqueAbilityInfo.json](API%20%EB%AA%85%EC%84%B8/uniqueAbilityInfo.json)
        
        이 부분은 cardBaseInfo처럼 추후 페이지네이션을 해야 할 수도 있음
        
        - unique_abilities
            
            고유 특수 효과 목록
            
            - unique_ability_id
                
                고유 특수 효과 식별 번호 **int**
                
            - unique_ability_name
                
                고유 특수 효과 이름 **string**
