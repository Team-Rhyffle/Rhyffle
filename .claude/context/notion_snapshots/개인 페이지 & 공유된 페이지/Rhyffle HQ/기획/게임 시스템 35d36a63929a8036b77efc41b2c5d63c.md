1. 탭 노트 (영상의 흰색 노트)
    1. 가장 기본적인 노트로, 노트가 도착하는 순간 화면을 터치해서 처리
2. 홀드 노트 (파랑색 선 노트)
    1. 시작 지점에서부터 터치를 유지하고, 끝나는 지점에서 터치를 뗌
    2. 기본적으로 시작과 끝 지점에 판정 1회씩이 존재하고, 보다 긴 홀드의 경우에는 곡의 BPM따라 사이 판정 (내부 노트)들이 추가로 존재함.
3. 슬라이드 노트 (파란색 노트)
    1. 노트가 지나가는 동안 누를 필요 없이 화면이 터치된 상태이면 처리
4. 플릭 노트 (초록/빨강 노트)
    1. 탭 노트 처리방식에 더해 누르는 순간 위/아래로 스와이프

Rhyffle 에서는 위 용어 및 대분류를 그대로 사용하되, 아래의 차이점이 있음.

1. 플릭 노트의 종류에 좌우플릭이 추가됨. 좌우플릭의 처리 방식은 상하플릭과 같은 x축 방향 거리만큼을 스와이프하면 처리됨. 거리 = {정하기}
2. 홀드 / 슬라이드 노트는 하늘색, 상하플릭은 빨강/초록, 좌우플릭은 노랑/보라 색 을 이용함.
    1. 해당 색상은 COSMO FCO의 다섯가지 Season에 대한 상징색. 원본 색상 사용해보고 채도가 너무 세다 싶으면 낮춰서 사용.

## 3. 판정

노트 당 4가지의 판정 범위가 존재함. 홀드 노트의 경우 각각의 내부 노트들에 별개로 판정 범위가 존재

![image.png](%EA%B2%8C%EC%9E%84%20%EC%8B%9C%EC%8A%A4%ED%85%9C/image.png)

노트 오브젝트의 세로 길이가 h, 배속 정도가 r일 때, (r = 1일 때 시스템 기본 배속. 0.1~3.0 까지 일단 지원)

Perfect (하늘색) 범위

- (1 + 0.2*r) * h
- 노트 점수의 120% 획득

Great (연두색)

- (1 + 0.4*r) * h
- 노트 점수의 100% 획득

Good (주황색)

- (1 + 0.7*r) * h
- 노트 점수의 70% 획득

Miss: 노트를 놓침

- (1 + r) * h
- 노트 점수의 0% 획득

노트별 세부 판정

- 홀드 노트 : 사이 판정 노트들 (긴 홀드노트의 경우) 이 존재하는 경우, 각각의 판정 노트들에 점수가 부여된다. 즉, 8개의 판정 노트로 이어진 홀드노트 중 4개의 판정 노트까지 판정을 받고 그 다음 끊겼다면, 5번째 판정 노트부터는 모든 판정이 fail 처리 된다.
    - 노트 개수는 홀드 노트 하나가 아닌 사이 판정 노트들의 합만큼 노트 개수를 차지한다.
    - 위 예시에서 8개의 판정 노트로 이어진 홀드 노트는, 노트 하나가 아니라 노트 8개라고 생각하면 됨
    - 콤보 숫자도 당연히 개별로 쌓임
- 플릭 노트 : 상하플릭, 좌우플릭 모두 일정 거리를 해당 방향으로 플릭해야 판정의 최소 조건을 충족하게 된다. (Perfect, Great, Good 등은 이 플릭이 만족된 이후 탭 노트와 동일한 기준으로 판정)
    - 일정 거리는 현재 판정선의 좌우 길이 (전체 가로 판정선이 아닌, 각 라인별 판정선) 의 1/3 이다.

## 4. 점수 시스템

### Plain Mode 기준 설명

- 1,000,000점을 기준으로, 노트의 총 개수로 나누어 노트별 점수가 책정됨.
    - 나누어진 값이 정수가 아닌 경우, 소수점 첫째자리에서 반올림 후 곡의 첫번째 노트에 남는 값 더하거나 뺌.
- ex) 총 노트가 1444개인 곡의 경우, 1000000 / 1444 = 692.52
    - 노트당 693점
    - 첫번째 노트는 1000000-693*1443 = 1점
        - 가장 극단적인 경우로, 다른 경우에서는 이정도로 편차 안남.
- 합산 점수는 노트 점수 * 판정 배율로 계산.
    - 이론상 만점 = 1,200,000점

### Challenge Mode 설명

- Plain Mode와 기본 노트 점수 계산까지는 모두 동일
- 클래스 점수 : 각 카드의 기본 점수 (카드의 class에 따라서 차등) 가 노트 점수에 더해진다. 아래는 그냥 예시
    - First / Basic : + 10점
    - Double / Event : + 20점
    - Special : + 30점
    - Unit : + 40점
    - Premier : + 50점
- 시너지 점수 : 특정 카테고리를 만족한 카드들이 필드의 8장 중 일부로 구성될 경우, 해당 필드는 시너지 점수를 발생시킨다. 시너지 점수는 종류마다 합연산으로 작동할수도, 곱연산으로 작동할수도 있다.
    - class
    - member
        - 해당 멤버가 참여한 음원에 대해서는 추가 시너지 점수_1 을 얻는다
        - 같은 멤버 카드가 n장 이상일 시 시너지 점수_2 를 얻는다
    - season
        - 해당 시즌에 발매된 음원에 대해서 추가 시너지 점수_1 을 얻는다
        - 같은 멤버 카드가 n장 이상일 시 시너지 점수_2 를 얻는다
    - theme (collection)
- 즉, 총 합산 점수는 { **Σ** (노트 점수 + 클래스 점수 + 시너지 점수_1) * (시너지 점수_2)} * 판정 배율 이 된다.

## 5. 카드 시스템

- 카드 : COSMO의 NFT 포토카드인 Objekt로, 다음의 구성 요소를 갖는다.
    - 이미지 : 포토카드의 전면 이미지. 가로:세로 1:1.54 비율을 갖는다. (정확히는 1083 x 1673)
    - 아티스트 (artist) : 어떤 아티스트의 (그룹의) 오브젝트인지 나타낸다.
    - 멤버 (member) : 어떤 멤버의 오브젝트인지 나타낸다.
    - 시즌 (season) : 오브젝트가 언제 발행되었는지 나타내는 값.
        - tripleS / ARTMS : Atom01 → Binary01 → Cream01 → Divine01 → Ever01 → Atom02 순으로 뒤 숫자가 높아지면서 5가지 시즌이 순환한다.
        - idntt : Spring25 → Summer25 → Autumn25 → Winter26 → Spring26 순으로 실제 사계절 이름 - 연도 대로 순환한다.
    - 컬렉션 (collection) : 숫자 3자리 + 알파벳 A 또는 Z 구조로 이뤄진 값. (ex. 123Z) 한 시즌 안에서 고유하다.
        - 숫자 - 맨 앞자리 (백의자리) 숫자가 class를 나타내긴 하는데, 그냥 세자리 통째로 시즌 안에서 고유값이라고 이해하는게 편함.
        - 알파벳 - A는 오프라인 실물카드가 쌍으로 존재하는 경우, Z는 디지털 NFT만 존재하는 경우
            - 오프라인 실물카드 뒷면의 QR을 스캔하면, COSMO NFT로 같은 이미지 포토카드가 민팅됨
    - 클래스 (class) : 포토카드의 등급. 등급이 높다고 꼭 더 희귀하지는 않다.
        - tripleS / ARTMS : First < Double < Special < Premier < Motion
        - idntt : Basic < Event < Special < Unit < Motion
        - 그 밖에 가입 선물인 Welcome, 이제는 안 나오는 Zero, 정말 한정적으로만 발매하는 tripleS / ARTMS의 Unit 등등 예외적인 Class들이 존재는 한다.

- 현재 코스모에는 미친듯이 많은 오브젝트 종류가 있어서, 이걸 데모에 다 구현하는건 불가능하고, 멤버 수와 시즌 수를 어느정도 고려해서 모든 구성 요소들이 distinct 하게 존재할 수 있게, 시너지 조건 매수를 맞출 수 있는 수준까지만 하드코딩된 json key-value 형식의 유사 DB를 사용할 계획.

## 6. 덱 시스템

### 구성 요소

- 덱 : 게임 시작 시 유저가 소유하는 카드 더미로, 40장으로 구성됨
    - 드로우가 진행되며 덱이 빈 경우, 묘지의 모든 카드를 셔플한 후 덱으로 이동시킨다 (리바이브)
- 드로우 : 덱에서 빈 필드의 공간에 카드를 놓는다
    - 게임 시작 시, 기본적으로 8개의 드로우 실행
    - 일정 시간/마디 수 마다 다음 8장의 set으로 교체된다.
        - 현재 필드의 8장을 모두 묘지로 이동 / 덱에서 다음 8장을 드로우
    - 셔플 키 : 카드 구역 왼쪽에 위치하며, 특정 콤보 게이지 이상을 누적 시 활성화된다.
        - 남은 시간과 관계 없이 ‘유저가 직접 클릭해서 활성화’ 시 새로운 드로우를 진행한다.
        - 만약 현재 판정이 진행되고 있는 상황이라면 (완전 동타이밍은 힘들겠지만) 판정이 끝난 즉시 새 드로우를 진행한다.
    - 새 드로우 하고 새로운 시너지를 계산 → 점수에 적용하는데 offset이 필요한가? 만약 필요하다면 일정 주기가 아닌 곡마다 노트 비는 시점에 드로우를 실행하는쪽으로 생각해야 할수도 있음.
- 필드 : 카드들이 드로우 된 후, 리듬 게임 점수에 효과를 발동하는 영역
    - 총 8개의 슬롯이 존재함
    - 덱, 묘지의 카드는 보너스 점수에 영향을 미치지 못하고, 오직 필드의 카드들만으로 계산이 이루어진다.
- 묘지 : 필드에서 소모된 카드가 이동하는 영역
    - ~~묘지의 구조는 FIFO를 따른다~~ 묘지 드로우 개념이 없으므로 아무렇게나 쌓여도 됨
