### 각 필드 설명

- 카드 이미지: 실제 코스모에서는 서버에 이미지파일을 저장해두고, DB에는 해당 링크 정보를 카드마다 갖고 있다. 현재 데모에서는 서버를 사용하지 않을 예정이므로, 로컬 파일명을 저장해두고 이걸 불러오는 식으로 대응하는게 좋을 것 같다.
- 그룹명: tripleS, idntt 등의 그룹 이름이다.
- 멤버명: SeoYeon, DoHun 등의 멤버 이름 영문명이다.
    - ~~멤버id: 그룹마다 1번부터 시작해서 멤버의 고유 id 숫자값이 존재한다. 멤버 이름이 그룹 안에서도, 그룹 간에도 겹치는 경우가 아직 없어서 쓸데없긴 하다.~~
- 컬렉션 번호: 한 시즌 안에서 몇번째, 어떤 클래스(등급) 으로 발행된 오브젝트인지 정보를 담고 있다. 숫자 세자리+A또는Z 알파벳의 4자리 string 값으로 구성된다.
    - 숫자 백의 자리는 희귀도, 남은 십의 자리/일의 자리 는 발행 순서를 나타내긴 하는데, 예외도 있다.
    - A는 오프라인 오브젝트도 존재하는 경우, Z는 온라인 오브젝트만 존재하는 경우이다.
- 시즌: 오브젝트가 발행된(출시된) COSMO상의 가상의 계절을 나타내는 값.
    - tripleS/ARTMS: Atom01 → Binary01 → Cream01 → Divine01 → Ever01 → Atom02 순으로 순환
    - idntt: Spring25 → Summer25 → Autumn25 → Winter25 → Spring26 순으로 순환
- 클래스: 오브젝트의 등급을 나타내는 값. 오른쪽으로 갈수록 높다.
    - tripleS/ARTMS: First → Double → Special → Premier
    - idntt: Basic → Event → Unit → Special
    - 그 밖에 Welcome, Zero, Motion 등등 몇 종류가 더 있지만 데모에서는 무시한다.
- 오브젝트번호: 앞서 설명했듯, 그룹명, 멤버명, 시즌, 컬렉션번호 4가지의 값이 조합되면 이는 오브젝트의 고유한 종류를 나타내게 되는데, 이 오브젝트 종류 중에서 몇번째로 발행된 오브젝트인지 나타낸다.
    - ex) ‘tripleS SeoYeon Atom01 101Z’ 라는 오브젝트 종류에서, #00123의 오브젝트 번호를 가지면, 그 중 123번째로 발행된 오브젝트라는 뜻.
    - 이 오브젝트번호까지 더해지면, 특정 유저가 보유한 고유한 오브젝트 객체 하나씩을 나타낸다.

## 2. 카드 강화 시스템

- Rhyffle의 카드 강화는 아래 두 가지 방법이 있다.
    - 화음 - 해당 카드와 동일한 아티스트, 동일한 등급, 다른 멤버의 카드로 강화한다.
    - 공명 - 해당 카드와 동일한 아티스트, 동일한 등급, 동일한 멤버의 카드로 강화한다.
- 화음 / 공명 모두 10단계까지 강화를 진행할 수 있다.
- 강화 실패 확률은 없다.

## 4. 카드 DB

### json파일 형식 - 카드 도감 DB

- 데모 제작을 위한 가상의 카드 DB로, COSMO Objekt 관련 API를 대체한다.

```json
{
	"Key": { // 아티스트이름 + 시즌이름 + 멤버이름 + 컬렉션번호의 고유조합.
		"imageFile": {string}, // 이미지파일의 경로
		"artist": {string}, // 아티스트 이름 (그룹명)
		"member": {string}, // 멤버 이름 (사람이름)
		"season": {string}, // 시즌 이름 + 숫자 (ex. Atom01)
		"collectionNo": {string}, // 컬렉션 번호. 번호이지만 뒤에 붙는 A/Z 때문에 string형임
		"class": {string} // 클래스 이름 (영문)
	}, //아래는 예시
  "tripleS Atom01 ChaeYeon 101Z": {
    "imageFile": "{이미지파일경로}\Atom01_ChaeYeon_101Z.jpg",
    "artist": "tripleS",
    "member": "ChaeYeon",
    "season": "Atom01",
    "collectionNo": "101Z",
    "class": "First"
  }
}
```

### json파일 형식 - 개별 카드 DB

- 위는 카드 종류의 DB이고, 여기는 개별 카드 객체 (실제로 유저가 보유중인) 에 대한 DB이다.

```json
[
  {
    "objektId": {int}, // [Primary Key] 이 카드의 절대 고유 식별자
    "userId": {int}, // [Foreign Key] 이 카드를 소유한 유저의 ID
    "objektName": {string}, // [Foreign Key] 카드 도감의 ID (어떤 종류의 카드인지 식별)
    "harmonyLevel": {int}, // 현재 화음 횟수
    "resonanceLevel": {int}, // 현재 공명 횟수
    "acquiredAt": {timestamp} // 카드를 획득한 시간
  },
  { // 예시
    "id": 123,
    "userId": 456,
    "objektName": "tripleS Atom01 NaKyoung 107Z",
    "harmonyLevel": 4,
    "resonanceLevel": 8,
    "acquiredAt": "2026-05-18T14:00:00Z"
  }
]
```
