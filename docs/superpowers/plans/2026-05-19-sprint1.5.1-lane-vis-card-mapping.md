> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Background (사용자 피드백, 2026-05-19):**
1. 현재 화면: Bar 24개 점 → 점 사이 띠 23개로 인식. 사용자는 lane = 띠로 보고 싶음 → Bar 25 점, 띠 24 lane.
2. CardBoard 슬롯이 lane 3개 폭보다 훨씬 큼 → 카드 1개 = 정확히 lane 3개 폭.
3. 24 lane / 3 = **카드 8장** 필요 (현 7장).
4. lane 클릭 시 그 위의 카드 슬롯 시각 hover **+** 그 카드의 효과만 해당 영역(lane 3개)의 노트 판정에 적용.

## 핵심 결정 (2026-05-19)

- **Bar = 점 marker, LaneAnchor = 입력 collider** (시각/입력 책임 분리). Bar 25개 = lane 24개 경계점.
- `BarX(n) = n - 13` (Bar_1 x=-12, Bar_25 x=+12) — Bar 25 중심정렬.
- `LaneX(line) = line - 11.5` (lane 0 center x=-11.5, lane 23 center x=+11.5) — 수치는 Sprint 1과 동일하므로 노트 trajectory 검증 그대로 통과 예상.
- LaneAnchor.size_x = `LANE_WIDTH = 1.0` (lane 폭 그대로). 25 Bar (size_x 0.32) + 24 LaneAnchor (size_x 1.0). 둘 다 BoxCollider.
- 입력 raycast 시 **LaneAnchor 우선** (Bar는 collider 없음 → raycast 안 잡힘).
- `LaneToCardIndex(lane) = lane / 3` (lane 0~2 → card 0, ..., lane 21~23 → card 7).
- `CardSystem.GetScoreMultipliersForSlot(cardIndex)` — 해당 slot의 effect만 modifier 반환. GameLoop는 `note.Line / 3`으로 카드 인덱스 계산 후 호출. 전체 합산이 아닌 영역 한정.
- CardBoardUI 슬롯 width = Canvas RefWidth (956) / 8 = **119.5 px** (3 lane 폭 매칭).
- Hover 하이라이트: GameLoop.HandlePress가 CardBoardUI에 `OnLaneHover(int lane)` 호출 (또는 EventBus `LaneHoveredEvent`). 단발성 highlight + N프레임 후 fade.
