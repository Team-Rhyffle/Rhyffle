## 핵심 결정 (2026-05-19)

**Figma 좌표 변환 (portrait → landscape 956×440)**

Figma frame "iPhone 16 Pro Max - 3" (id 2:16): metadata는 portrait 좌표로 작성됐고 시각은 landscape로 90도 회전 표시. 변환 규칙:
- Portrait `(x, y, w, h)` → Landscape `(y, 440 - x - w, h, w)` (counter-clockwise 90도)

각 영역 landscape 좌표 (Canvas RefRes 956×440 기준, anchor top-left=(0,0)):

| 영역 | Portrait (x, y, w, h) | Landscape (x, y, w, h) |
|---|---|---|
| 노트/라인 영역 | (0, 119, 345, 718) | (119, 95, 718, 345) |
| 판정선 영역 | (345, 119, 30, 718) | (119, 65, 718, 30) |
| 카드 영역 | (375, 119, 65, 718) | (119, 0, 718, 65) |
| 곡 정보 | (78, 837, 119, 129) | (837, 243, 129, 119) |
| 셔플 | (361, 876, 80, 79) | (876, -1, 79, 80) |
| 덱묘지 | (361, 0, 83, 79) | (0, -1, 79, 83) |
| 현재 점수 | (-1, -4, 123, 79) | (-4, 318, 79, 123) |
| 콤보 수 | (78, -4, 123, 79) | (-4, 239, 79, 123) |
| 일시정지 (frame 외부) | n/a — Figma absolute (1351, 399, 77, 79); frame origin (2230, 399) — landscape (-879, 0, 77, 79) → frame 밖 |

**Note**: 좌표 변환이 약간 어색한 영역들이 있음. 일시정지가 frame 밖에 그려진 게 디자인 의도인지 placeholder 실수인지 불확실. **Sprint 1.5.3에선 일시정지를 frame 안 좌상단으로 배치** (Lo-Fi 시각 추정 + UX 상식).

**Canvas anchor 변환** (Unity Canvas top-left = (0, 440)):
Lo-Fi의 top-left 기준 (x, y) → Unity Canvas 좌상단 anchor 기준 (x, -y).

**적용 정책**:
- 모든 신규 placeholder는 회색 또는 노란 반투명 Image + 라벨 TextMeshProUGUI
- 노트/라인 영역, 판정선 영역은 lane GameObjects 자체와 별개 visual band — Canvas overlay로 박스 표시 (lane GameObjects는 world space, 그대로 유지)
- 카드 영역은 별도 visual 박스 — CardBoardUI (World Space)는 그대로
- Score/Combo는 기존 TextMeshProUGUI를 이동만 (재생성 X)
- JudgementText는 중앙 유지 (Lo-Fi에서 별도 영역 미정 — 노트 영역 위 overlay로 가정)
