# Sprint 1.5 — Card Effect Framework + Lo-Fi UI Placement Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Sprint 1 plain mode 위에 카드 효과 발동 프레임워크 + 카드 placeholder UI를 얹어, **3장 dummy card가 실제 노트 판정에 영향을 미치는 것을 검증한다**. 카드 데이터셋(≈400장)이나 일반능력 random 발급은 본 phase 미포함 — 뼈대 + 실제 작동 증명만.

**Architecture deltas (Sprint 1 → 1.5):**
- 추가: `EventBus<T>` (publish/subscribe), `JudgmentResult` struct (현재 grade enum → struct + context), `CardData` DTO, `CardEffect` interface + `CardEffectRegistry`, `CardSystem` MonoBehaviour (필드 7장 관리 + 발동), `PokerHandEvaluator` (stub — 원페어만), `CardBoardUI` (Canvas 자식 7 슬롯)
- 수정: `GameLoop.ApplyJudgment` — 직접 score 갱신 → `EventBus.Publish<NoteJudgedEvent>`. `CardSystem`이 구독하여 카드 효과로 score/combo modifier 반환. GameLoop가 최종 적용
- 미수정: GameConfig / Conductor / ChartLoader / Note / Bar / NoteScreen — Sprint 1 그대로

**Tech stack:** Sprint 1 동일 (Unity 6000.2.8f1 / Built-in / C# / JsonUtility). 신규 의존성 0.

3. **per-note-hit / combo-milestone hook**: 본 framework가 신설. `NoteJudgedEvent`, `ComboChangedEvent` 이벤트 정의. 카드 효과가 구독
4. **일반능력 random 발급**: 본 phase skip. 더미 카드는 고유능력만
