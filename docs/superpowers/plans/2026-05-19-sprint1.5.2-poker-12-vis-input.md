## 핵심 결정 (2026-05-19)

**Poker hand 정의 (사용자 확정)**:
- 원페어(1) — same rank ×2
- 투페어(2) — same-rank pair ×2
- 트리플(3) — same rank ×3
- 스트레이트(4) — 5 consecutive ranks (A=1 또는 A=14 둘 다 허용? 일단 A=1만 — Sprint 2 결정)
- 플러시(5) — same suit ×5
- 쓰리페어(6) — same-rank pair ×3
- 풀하우스(7) — 트리플 + 페어
- 레인보우(8) — 4 suit 모두 등장 (각 ≥1)
- 더블트리플(9) — same rank ×3 그룹 ×2
- 포카드(10) — same rank ×4
- 럭키세븐(11) — 7 consecutive ranks (필드 7장 모두 connected, A=1 only)
- 스트레이트플러시(12) — 5 consecutive ranks + same suit
- 없음(0) — 위 어느 것도 아님

**우선순위**: enum 값 큰 게 높음. Evaluate는 12 → 1 순으로 검사, 첫 매칭 반환.

**A=1 / A=14 처리**: Sprint 1.5.2에선 **A=1 만 허용** (enum CardRank.A = 1). A-high 스트레이트 (10-J-Q-K-A)는 Sprint 2 spec 확정 시. 어차피 럭키세븐 = 필드 7장 모두 connected → A-K로는 7장 모두 빈틈없이 불가능 (13 rank 중 7 connected = A-7 한 가지). 럭키세븐은 A-7 (rank 1-7) 한 패턴만 매칭.

**Visual 개선**:
- Bar: 1×1 white sprite (alpha 0.3) → **작은 원형 점 marker** (alpha 0.6, 크기 0.15 unit, 색 흰색 또는 약한 청회색). placeholder 수준이지만 점이 더 명확.
- LaneAnchor: 현재 visible sprite 없음 → **lane 띠 visible sprite** (alpha 0.1 회색, 띠 영역 24 lane 각각). 너무 진하면 노트 가독성 ↓.
- Hover highlight: 마우스 hover시 (실 입력 + 단순 hover 둘 다) lane 띠 색 alpha 0.4로 boost + CardBoardUI.OnLaneHover 호출.

**실 입력 검증**:
- autoPerfectDebug=false 시 lane 클릭 → 노트 판정 동작 확인 (Sprint 1.5.1 T5에서 코드는 수정했으나 실제 검증은 안 됨)
- dummy_test.json 채보 그대로 사용 가능
- Unity Play mode 진입 + 사용자 수동 클릭 검증 (자동 ticking으론 입력 시뮬레이션 불가)
